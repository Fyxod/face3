# FACE3 A6000 instructions

FACE3 optimizes:

```text
Z = cosine_similarity(ArcFace(original_edit), ArcFace(perturbed_edit))
loss = Z
```

The perturbed edit is generated by a differentiable InstructPix2Pix forward pass inside the optimization loop. InstructPix2Pix and ArcFace weights are frozen. Only perturbation parameters are updated.

This is heavier than FACE/FACE2. Start with the smoke commands exactly as written.

Important memory note:

```text
edit_steps=2  -> timing/autograd smoke only; edits may look bad.
edit_steps=20 -> visually closer to normal InstructPix2Pix, but full backprop can use very high VRAM.
```

FACE3 now enables editor gradient checkpointing by default. If a run still OOMs at `--edit-steps 20`, lower `--edit-steps` to `8` or `12` before attempting longer optimization.

## 2. Differentiable edited-output smoke

Before optimization timing, run the stock-vs-differentiable parity check. It compares the normal diffusers `pipe(...)` edit against FACE3's differentiable wrapper using the same image, prompt, seed, and settings.

```bash
cd /home/interns/Desktop/face3

$HOME/.local/bin/micromamba run \
  -p /home/interns/Desktop/mat/.micromamba/envs/mat-a6000 \
  python -m face3.scripts.check_instruct_parity \
  --mat-root /home/interns/Desktop/mat \
  --face-id face_002 \
  --prompt "add black sunglasses" \
  --seed 250852 \
  --edit-steps 20 \
  --output-root outputs/instruct_parity
```

Inspect:

```text
outputs/instruct_parity/.../parity_sheet.png
```

If the stock edit is good but the differentiable wrapper edit is not, stop and push `outputs/instruct_parity`.

Use only 1 optimization iteration and 2 InstructPix2Pix edit steps first:

```bash
cd /home/interns/Desktop/face3

$HOME/.local/bin/micromamba run \
  -p /home/interns/Desktop/mat/.micromamba/envs/mat-a6000 \
  python -m face3.scripts.smoke_identity \
  --mat-root /home/interns/Desktop/mat \
  --arcface-checkpoint /home/interns/Desktop/face3/models/arcface/iresnet100.pth \
  --iters 1 \
  --edit-steps 2 \
  --geometry-config configs/geometry_default.json
```

Expected output root:

```text
outputs/smoke_dct_image_pipeline/
```

This checks:

- ArcFace checkpoint loads
- InstructPix2Pix loads
- the clean edit reference is generated
- the perturbed edit is generated under autograd
- `Z = cosine_similarity(ArcFace(original_edit), ArcFace(perturbed_edit))` is finite
- `loss = Z` is finite
- gradients reach perturbation parameters
- no `.pt` replay tensors are saved

If this fails, stop and push logs.
